Reports expect declarations that are missing an actual implementation on one or more of the project's platforms.

The Kotlin compiler only verifies expect/actual pairing for the target you are currently building. This inspection checks all platforms observed in the project at once, using the same project-wide scan as the Expect/Actual tool window.

The quick fix generates an actual stub for the missing platform: it places a TODO-bodied declaration in the platform's source set, in the same package and file name as the expect.

Platforms are detected from source-set directory names (e.g. androidMain, iosMain) with a module-name fallback. The set of "known platforms" is the union of every platform that has at least one actual anywhere in the project.
